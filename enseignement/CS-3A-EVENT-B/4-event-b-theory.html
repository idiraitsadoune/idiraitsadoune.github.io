The Event-B method

CONTEXT $ctx_1$
EXTENDS $ctx_2$

SETS $s$
CONSTANTS $c$
AXIOMS
	$A(s,c)$
THEOREMS
	$T(s,c)$
END

MACHINE $mch_1$
REFINES $mch_2$
SEES $ctx_i$

VARIABLES $v$
INVARIANTS
	$I(s,c,v)$
THEOREMS
	$T(s,c,v)$
EVENTS
	$[events\_list]$
END

$event$ $\mathrel{\widehat=}$
	any $x$
	where
		$G(s,c,v,x)$
	then
		$BA(s,c,v,x,v')$
	end
$A(s,c) ~~~\vdash~~~ T(s,c) $
$A(s,c) \land I(s,c,v) ~~~\vdash~~~ T(s,c,v)$
$A(s,c) \land I(s,c,v) \land G(s,c,v,x) ~~~\vdash~~~ \exists v'.BA(s,c,v,x,v')$
$A(s,c) \land I(s,c,v) \land G(s,c,v,x) \land BA(s,c,v,x,v') ~~~\vdash~~~ I(s,c,v')$
$...$

Limites du langage mathématique natif

  • Types prédéfinis ($\mathbb Z$, $BOOL$, ensembles)
  • Opérateurs standards
  • Difficulté à modéliser :
    • des structures de données spécifiques
    • des domaines métiers complexes
    • des calculs spécialisés (temps, graphes, files, etc.)

Motivation

  • Réutilisabilité des concepts mathématiques
  • Modularité et abstraction
  • Extension contrôlée du langage Event-B

Qu'est-ce qu'une théorie ?

Une théorie est un module définissant :
  • des types abstraits
  • des opérateurs
  • des axiomes
  • des théorèmes
  • des règles de preuve
Positionnement dans Rodin
  • Intégration au niveau des contextes/machine
  • Impact sur la génération des obligations de preuve
  • Interaction avec les prouveurs automatiques et interactifs

The theory plugin

  • Theory Plug-in provides capabilities to extend the Event-B mathematical language and the Rodin proving infrastructure.
  • An Event-B theory can contain :
    • new datatype definitions,
    • new polymorphic operator definitions,
    • axiomatic definitions,
    • theorems,
    • associated rewrite and inference rules.
  • Michael J. Butler and Issam Maamria.
    Practical theory extension in Event-B. Theories of Programming and Formal Methods. 2013.
  • Thai Son Hoang, Laurent Voisin, Asieh Salehi, Michael J. Butler, Toby Wilkinson, and Nicolas Beauger.
    Theory plug-in for Rodin 3.x. CoRR, abs/1701.08625, 2017.

Types abstraits (Abstract Types)

Un type abstrait est un type :
  • sans implémentation concrète
  • aucune hypothèse sur sa représentation interne
  • défini uniquement par ses propriétés (axiomes)
  • manipulé via des opérateurs
  • On raisonne sur ce que c'est, pas sur comment c'est implémenté.
THEORY $REAL$
AXIOMATIC DEFINITIONS
  TYPES
    $\mathbb R$
	OPERATORS
		zero : $\mathbb R$
    one : $\mathbb R$
    plus(a $\in$ $\mathbb R$, b $\in$ $\mathbb R$) : $\mathbb R$ INFIX
    sub(a $\in$ $\mathbb R$, b $\in$ $\mathbb R$) : $\mathbb R$ INFIX
    mult(a $\in$ $\mathbb R$, b $\in$ $\mathbb R$) : $\mathbb R$ INFIX
    div(a $\in$ $\mathbb R$, b $\in$ $\mathbb R$) : $\mathbb R$ INFIX
      well-definedness condition: b $\neq$ zero
    ...
    lt(a $\in$ $\mathbb R$, b $\in$ $\mathbb R$) : $BOOL$ INFIX
    ...
	AXIOMS
		$\forall$ x,y $\cdot$ (x plus y $=$ y plus x)
    $\forall$ x $\cdot$ (x plus zero $=$ x)
    ...
    $\forall$ x $\cdot$ (x mult zero $=$ zero)
    ...
END

Les types inductifs

Un type inductif est défini par :
  • un type abstrait
  • un ou plusieurs constructeurs
  • des axiomes d'induction et de disjonction
  • éventuellement des fonctions définies par récursion
THEORY $List$
TYPE PARAMETERS
  $T$
DATATYPES
  List($T$) $\mathrel{\widehat=}$
    $\triangleright$ nil
    $\triangleright$ cons(head $\in$ $T$, tail $\in$ List($T$))
OPERATORS
  append(l $\in$ List($T$), x $\in$ $T$) PREFIX $\mathrel{\widehat=}$
    recursive definition
      append(nil, x) $\mathrel{\widehat=}$ cons(x, nil)
      append(cons(y, k), x) $\mathrel{\widehat=}$ cons(y, append(k, x))   
  listsize(l $\in$ List($T$)) PREFIX $\mathrel{\widehat=}$
    recursive definition
      listsize(nil) $\mathrel{\widehat=}$ $0$
      listsize(cons(x, k)) $\mathrel{\widehat=}$ $1$ $+$ listsize(k)
  ...
THEOREMS
  @thm1: $\forall$ l,x $\cdot$ (l $\in$ List($T$) $\land$ x $\in$ $T$ 
              $\Rightarrow$ listSize(append(l,x)) $=$ $1$ $+$ listSize(l))
  ...
END

Rôle des règles de preuve
dans une théorie

Dans Event-B, la majorité des obligations de preuve sont traitées par :
  • des prouveurs automatiques
  • des règles mathématiques connues
Le plugin THEORY permet d'ajouter :
  • des règles de réécriture $\to$ remplacer une expression (simplification)
  • des règles d'inférence $\to$ déduire un fait (raisonnement logique)

Règles de réécriture
(Rewrite Rules)

Une règle de réécriture permet de remplacer automatiquement une expression par une autre équivalente 
THEORY $REAL$
AXIOMATIC DEFINITIONS
  TYPES
    $\mathbb R$
PROOF RULES
  add_com
    Metavariables
      a $\in$ $\mathbb R$
      b $\in$ $\mathbb R$
    Rewrite Rules
      rew1 : a plus b 
      rhs1 : T $\vdash$ b plus a 
  ...
END

THEORY $REAL$
AXIOMATIC DEFINITIONS
  TYPES
    $\mathbb R$
PROOF RULES
  ...
  div_one
    Metavariables
      x $\in$ $\mathbb R$
    Rewrite Rules
      rew1 :  x div x 
      rhs1 : x $\neq$ zero $\vdash$ one 
  ...
END

Règles d'inférence
(Inference Rules)

Une règle d'inférence permet de déduire une conclusion à partir de prémisses
$$P_1, P_2, ..., P_n ~~~\vdash~~~ C$$ 
THEORY $REAL$
AXIOMATIC DEFINITIONS
  TYPES
    $\mathbb R$
PROOF RULES
  lt_transitivity
    Metavariables
      a $\in$ $\mathbb R$
      b $\in$ $\mathbb R$
      c $\in$ $\mathbb R$
    Inference Rules
      a lt b (in hypothesis)
      b lt c (in hypothesis)
      --------
      a lt c
  ...
END

THEORY $REAL$
AXIOMATIC DEFINITIONS
  TYPES
    $\mathbb R$
PROOF RULES
  ...
  leq_antisymetry
    Metavariables
      a $\in$ $\mathbb R$
      b $\in$ $\mathbb R$
    Inference Rules
      a leq b (in hypothesis)
      b leq a (in hypothesis)
      --------
      a = b
  ...
END

Apport fondamental du plugin THEORY

Le plugin THEORY permet de :
  • définir des types abstraits et inductifs
  • introduire des opérateurs spécialisés
  • formaliser la sémantique par des axiomes et des théorèmes
  • guider les prouveurs grâce à des règles de réécriture et d'inférence
