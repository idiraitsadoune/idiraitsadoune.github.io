Description du problème

Jeu du Tic-tac-toe

Le tic-tac-toe, aussi appelé morpion, est un jeu de réflexion se pratiquant à deux joueurs, tour par tour, dont le but est de créer le premier un alignement.

Deux joueurs s'affrontent. Ils doivent remplir chacun à leur tour une case d'une grille $3 \times 3$ avec le symbole qui leur est attribué : O ou X.
Le gagnant est celui qui arrive à aligner trois symboles identiques, horizontalement, verticalement ou en diagonale.
Il est coutume de laisser le joueur jouant X effectuer le premier coup de la partie.

Nous souhaitons développer une version du jeu en utilisant une architecture Client/Serveur. Deux clients doivent se connecter pour pouvoir lancer le jeu. Le premier client joue avec le symbole X et le second avec le symbole O.
Le serveur gère le jeu en envoyant les messages suivants aux clients :

  • "play": pour demander à un cleint de de jouer une case
  • "invalide-position": pour demander au client de re-jouer une nouvelle case (car la précédente case jouée est déjà occupée ou n'existe pas)
  • "win": pour informer le client de la fin de la partie par une victoire
  • "lose": pour informer le client de la fin de la partie par une défaite
  • "tie": pour informer le client de la fin de la partie sans vainqueur
Le serveur envoie également l'état de la grille après chaque message de type "play" pour permettre au joueur de jouer le bon coup.

Côté client, selon le type du message reçu, une réaction doit être programmée :

  • à la réception de "play": attendre la réception de la grille
  • à la réception de la grille : afficher la grille, demander la saisie des coordonnées de la case à jouer et les envoyer avec le symbole joué au serveur
  • à la réception de "invalide-position": re-demander la saisir des coordonnées de la case à jouer et les renvoyer avec le symbole joué au serveur
  • à la réception de "win": afficher un message de victoire
  • à la réception de "lose": afficher un message de défaite
  • à la réception de "tie": afficher un message de fin de partie sans victoire/défaite

Travail à faire

  • Créer trois packages client, serveur et serveur.tests
  • Développer les classes Java nécessaires pour la gestion du Tic-tac-toe
    • pour le bon déroulement du jeu côté serveur, prévoir une méthode check_aligned_symbols qui prend en pramètre la grille et répond par true si trois symboles identiques sont alignés.
    • écrire quatre tests JUnits qui permettent de vérifier la méthode check_aligned_symbols dans les quatres cas suivants:
      • mêmes symboles alignés sur une ligne vérticale
      • mêmes symboles alignés sur une ligne horizontale
      • mêmes symboles alignés sur une diagonale
      • pas de combinaison gagnante
Barême

  • Code du serveur correct : 4 pts
  • Code du test correct : 4 pts
  • Code du client correct : 4 pts
